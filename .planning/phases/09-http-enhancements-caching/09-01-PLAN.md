```javascript
  const body = String(payload);
  res.writeHead(code, {
    ...SECURITY_HEADERS,

    'content-length': Buffer.byteLength(body, 'utf8')
  });

```

**downloadJson() at line 124 -- replace entire function with:**
```javascript
function downloadJson(res, filename, payload) {
  const body = JSON.stringify(payload, null, 2);
  res.writeHead(200, {
    ...SECURITY_HEADERS,
    'content-type': 'application/json; charset=utf-8',
    'content-disposition': `attachment; filename="${filename}"`,
    'content-length': Buffer.byteLength(body, 'utf8')
  });
  res.end(body);
}
```

CRITICAL: Use `Buffer.byteLength(body, 'utf8')` NOT `body.length`. German Umlauts (ae=U+00E4, oe=U+00F6, ue=U+00FC) are 2 UTF-8 bytes but 1 JS string character. Using body.length would send wrong Content-Length and cause ERR_CONTENT_LENGTH_MISMATCH in browsers.

Do NOT touch servePage() or serveStatic() -- those use fs.createReadStream().pipe(res) and should NOT have Content-Length (chunked transfer encoding handles framing for streams).
