Create `dvhub/core/auth.js`:

Import `fp` from `fastify-plugin` and `crypto` from `node:crypto`.

Export default a `fp`-wrapped async plugin function `authPlugin(fastify, opts)` where opts contains `{ apiToken, roles }`.

The plugin:
1. Calls `fastify.decorateRequest('userRole', null)` to add the property.
2. Adds a `preHandler` hook that:
   a. If no `apiToken` in opts: sets `request.userRole = 'admin'` and returns (open access mode).
   b. Extracts token from `request.headers.authorization?.replace('Bearer ', '')` OR `request.query?.token`.
   c. If no token found: `reply.code(401).send({ error: 'Authentication required' }); return;`
   d. Uses `crypto.timingSafeEqual` for token comparison. Both buffers created via `Buffer.from(string)`. If lengths differ OR timingSafeEqual returns false: `reply.code(401).send({ error: 'Invalid token' }); return;`
   e. Resolves role: if `opts.roles` is a Map/Object, look up token in it. If found, use that role. If not found but token matches apiToken, default to `'user'`. If opts.roles is not provided, default to `'admin'`.
   f. Sets `request.userRole = resolvedRole`.

The role hierarchy constant is also exported: `export const ROLE_HIERARCHY = { readonly: 0, user: 1, admin: 2 };`

A helper `export function hasRole(userRole, requiredRole)` returns `ROLE_HIERARCHY[userRole] >= ROLE_HIERARCHY[requiredRole]`.

Write tests FIRST using Fastify's `inject()` method for request simulation.
