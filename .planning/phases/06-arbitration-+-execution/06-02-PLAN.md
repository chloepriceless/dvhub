    **File 1: `dvhub/modules/exec/plan-intent-bridge.js`**

    Export `createPlanIntentBridge({ planEngine, eventBus, log })`.

    Subscribe to `planEngine.getActivePlan$()`. On each new plan:
    1. If plan is null: call `eventBus.emit({ type: 'control:intent', source: 'optimizer', priority: 4, action: 'clear', targets: {}, reason: 'plan_cleared', timestamp: Date.now() })` -- this signals the arbitrator to clear optimizer intents.
    2. If plan has slots: find the current slot where `Date.now() >= new Date(slot.start).getTime() && Date.now() < new Date(slot.end).getTime()`.
    3. If current slot found: emit `control:intent` with source 'optimizer', priority 4, action 'set', and targets extracted from slot fields: `{ gridSetpointW, chargeCurrentA, minSocPct }` (only include fields that are defined and not null in the slot).
    4. Set a setTimeout (with `.unref()`) for the next slot boundary to re-evaluate.

    On plan change: cancel any pending slot boundary timer, clear optimizer intents from arbitrator via intent with `action: 'clear'`.

    Return `{ destroy() }` that unsubscribes and clears timers.

    **File 2: `dvhub/modules/exec/index.js`**

    Export `createExecModule()` returning module contract `{ name: 'exec', requires: ['gateway'], init, destroy }`.

    `init(ctx)`:
    1. Create arbitrator: `const arbitrator = createArbitrator({ log: ctx.log })`
    2. Create executor: `const executor = createExecutor({ hal: ctx.hal, db: ctx.db, eventBus: ctx.eventBus, log: ctx.log })`
       - Get `hal` from `ctx.registry.get('gateway').hal` (gateway exposes hal on its return object)
    3. Subscribe to `ctx.eventBus.on$('control:intent')`:
       - On each intent: `const result = arbitrator.submitIntent(intent)`
       - If intent has `action: 'clear'` and `source`: call `arbitrator.clearSource(intent.source)`, return
       - For each target in `result.applied`: call `executor.executeCommand({ source: intent.source, priority: intent.priority, target, value: intent.targets[target], reason: intent.reason })`
       - For overridden targets: log at info level: `"Intent overridden: {source} {target} by {overriddenBy}"`
    4. If optimizer module is available (`ctx.registry.get('optimizer')`):
       - Get planEngine from optimizer module
       - Create plan-intent bridge: `createPlanIntentBridge({ planEngine, eventBus: ctx.eventBus, log: ctx.log })`
    5. Store subscription and bridge for cleanup
    6. Return `{ arbitrator, executor }` (exposed for route handlers and testing)

    `destroy()`: unsubscribe from event bus, destroy plan-intent bridge if created.

    Follow fp-wrapped Fastify plugin pattern matching DV module (see dvhub/modules/dv/index.js for the pattern). The module registers its Fastify plugin in init().
